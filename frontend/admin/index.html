Admin

Operator tools and diagnostics for BalanceWhiz. Restrict access to this URL in production (for example via your host’s IP allowlist, VPN, or HTTP basic auth in front of the app).

Test your API (step by step)

  1. Find your API address below. For BalanceWhiz production, that should be https://family-cash-flow-api.onrender.com/api/debug/public-config (full line under “Full check URL”). When the site was built, GitHub stored the backend root as API_BASE — not the literal __API_BASE__. If the URL is wrong, fix the secret in GitHub → repo → Settings → Secrets and variables → Actions, then re-run the Pages deploy workflow. Open this admin page at https://balancewhiz.com/admin/ after deploy.
  2. Click “Run API check”. Your browser asks the server for a small public JSON file (/api/debug/public-config). You do not need to log in.
  3. Read the green “What this means” box. That explains CORS, cookies, and whether the contact form can send email. If you see a red error instead, the browser could not reach the API (wrong URL, CORS, or network).
